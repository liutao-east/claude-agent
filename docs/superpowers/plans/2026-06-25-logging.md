# 后端日志（访问日志 + 业务追踪日志）实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 `server.py` 添加 HTTP 访问日志中间件和 `/ask`、`/ask_stream` 业务追踪日志，全部输出到控制台，沿用现有文本格式。

**Architecture:** 两类日志分离——`access` logger 由一个 `@app.middleware("http")` 统一记录所有请求的方法/路径/状态码/耗时；`code-qa` logger（现有）在 `/ask` 和 `/ask_stream` handler 末尾记录场景/会话/turns/cost/成败。日志行格式化抽取为两个纯函数（`_format_access`、`_format_turn`），便于单元测试；中间件与 handler 接入用 `TestClient` + `caplog` 做集成测试，mock 掉真实 SDK 与存储写入。

**Tech Stack:** Python 3.10+、FastAPI、标准库 `logging`、pytest（新增 dev 依赖）、httpx（TestClient 依赖，新增 dev 依赖）、uv（包管理）。

```python
# 访问日志中间件：记录所有 HTTP 请求的方法/路径/状态码/耗时。
@app.middleware("http")
async def logging_middleware(request: Request, call_next):
    start = time.perf_counter()
    response = await call_next(request)
    elapsed_ms = int((time.perf_counter() - start) * 1000)
    access_logger.info(
        _format_access(request.method, request.url.path, response.status_code, elapsed_ms)
    )
    return response
```

```python
def _format_turn(
    endpoint: str,
    *,
    scenario: str,
    conv_id: str | None,
    turns: int,
    cost: float,
    ok: bool,
    error: str | None,
) -> str:
    """格式化一条业务追踪日志正文。

    endpoint 为接口名（'ask' / 'ask_stream'）；conv_id 取前 8 位便于阅读；
    cost 格式化为 $0.0000；成功以 ok 结尾，失败以 error=<详情> 结尾。
    """
    short_conv = conv_id[:8] if conv_id else "-"
    cost_str = f"${cost:.4f}"
    if ok:
        return f"{endpoint} scenario={scenario} conv={short_conv} turns={turns} cost={cost_str} ok"
    return f"{endpoint} scenario={scenario} conv={short_conv} turns={turns} cost={cost_str} error={error}"
```
