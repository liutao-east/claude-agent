# 前端交互与功能说明（评审稿）

> 配套原型：项目根目录 `prototype.html`，双击用浏览器打开即可预览风格与交互（内置假数据，不连后端）。
> 本文只描述「要实现什么、怎么交互」，不涉及最终技术选型细节。

## 1. 总体目标

给现有的代码问答后端配一个**可爱风格的单页 Web 界面**，让用户能够：

1. 看到支持的场景，选择一个场景开始对话。
2. 查询本浏览器里的历史会话，点开继续追问。
3. 提问等待时看到友好的「思考中」反馈，避免误以为后端卡死。
4. 整体走柔和马卡龙 + 圆角 + emoji 的可爱风格。

## 2. 页面布局

```
┌─────────────┬───────────────────────────────────────┐
│  侧边栏       │  顶部：当前会话的场景徽章 🏷            │
│  (约 260px)  ├───────────────────────────────────────┤
│             │                                       │
│  🍡 标题/Logo │   消息流区域                            │
│             │   - 用户气泡（靠右，奶油色）              │
│  ＋ 新对话    │   - 助手气泡（靠左，白底，Markdown 渲染） │
│             │   - 等待时：思考中动画 + 计时             │
│  历史会话     │                                       │
│  · 会话A 🏷   │                                       │
│  · 会话B 🏷   ├───────────────────────────────────────┤
│  · 会话C 🏷   │   底部输入区：多行输入框 + 发送按钮 ✈️    │
└─────────────┴───────────────────────────────────────┘
```

- **侧边栏**：可爱标题、「＋ 新对话」按钮、历史会话列表（每项显示标题 + 场景徽章 + 相对时间，悬停出现删除按钮）。
- **主区域顶部**：显示当前会话锁定的场景徽章。
- **主区域中部**：消息流。空状态时这里改为「场景选择卡片」。
- **主区域底部**：输入框 + 发送按钮；回答进行中输入框禁用。

## 3. 功能与交互细节

### 3.1 选择场景对话（需求 1）

- 进入「新对话」或首次打开时，主区域中部展示**场景卡片**（数据来自后端 `GET /scenarios`）。
- 每张卡片显示：场景名、描述。点击即选中该场景并进入提问状态。
- 一旦该会话发出第一个问题，**场景被锁定**，后续追问不可更换（与后端约束一致）。顶部场景徽章常驻提示当前场景。

### 3.2 历史会话（需求 2）

- 历史**只记录本浏览器**（localStorage），不区分用户、不跨设备。
- 每次成功开启新会话，把会话信息写入 localStorage；后续每轮追问更新其时间。
- 侧边栏按最近活动倒序展示历史。
- 点击某条历史 → 调 `GET /conversations/{id}/messages` 加载完整历史消息并渲染，可继续追问。
- 删除：从 localStorage 移除该条（后端数据不动）。
- 若点开的会话在后端已不存在（404）→ 弹出可爱提示，并支持从列表移除该失效项。

**localStorage 结构**（键 `codeqa.sessions`）：
```json
[
  { "id": "会话UUID", "scenario": "code-qa", "title": "首次提问的前若干字…", "createdAt": "...", "updatedAt": "..." }
]
```

### 3.3 进度与推理反馈（需求 3）

- **不展示**真实工具调用或思考文本（后端不改）。
- 发送问题后、收到第一个回答字符前：显示**「思考中…」可爱动画**（跳动的马卡龙圆点）+ **已用秒数计时**。
- 回答开始后：内容**逐字流式**出现在助手气泡里（依赖后端现有的流式文本事件），用户能直观看到「它在认真答」。
- 回答结束：底部低调显示本轮的耗费/轮数等小信息（可选，不抢视觉）。
- 出错：在助手气泡里展示**可爱的错误提示**（例如「呜…出了点小状况，再试一次好吗？🥺」+ 具体错误）。

### 3.4 回答渲染

- 助手回答按 **Markdown** 渲染：代码块高亮、行内代码（如 `file.py:42`）、标题、列表、表格等都正常显示。
- 用户消息按纯文本显示（保留换行）。

### 3.5 可爱风格

- 配色：柔和马卡龙（粉、薄荷绿、奶油黄、淡紫），大圆角，柔和阴影。
- 适度 emoji 点缀（标题、按钮、空状态、错误提示）。
- 轻动画：思考中圆点跳动、气泡淡入、按钮 hover 微缩放。
- 保持事务化、可读，不幼稚；代码块用清晰的等宽字体与高亮主题。

## 4. 与后端的对接点（现有接口，无需改造）

| 用途 | 接口 |
| --- | --- |
| 列出场景 | `GET /scenarios` |
| 流式提问/追问 | `POST /ask_stream`（SSE：`token` / `done` / `error`；`conversation_id` 为空=新会话） |
| 加载某会话历史 | `GET /conversations/{id}/messages` |

> 说明：`/ask_stream` 是 POST，浏览器原生 `EventSource` 只支持 GET，因此前端用 `fetch` 读取响应流并手动解析 SSE。最终页面建议由后端**同源托管**（访问 `http://<host>:<port>/`），避免跨域配置。

## 5. 主要状态与边界

- **空状态**：无任何会话时，主区域展示场景卡片 + 一句可爱欢迎语。
- **加载中**：思考动画 + 计时；输入框禁用，发送按钮变「停止/等待」态。
- **流式中**：助手气泡逐字增长，自动滚动到底部。
- **错误**：网络断开、后端 500/502、SSE error 事件，统一显示可爱错误气泡，允许重试。
- **失效历史**：localStorage 中的会话后端已删除 → 提示并支持移除。

## 6. 暂不做（YAGNI）

- 不改后端 SSE / 不展示真实工具调用与思考过程。
- 不做用户登录、不做跨设备同步。
- 不做后端会话标题持久化（标题留在前端）。
- 不引入构建链 / SPA 框架（保持轻量）。
